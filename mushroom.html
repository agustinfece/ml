Context

It is a classification problem that is about predicting if a mushroom is edible or not. For this, UCI Mashroom Dataset will be used, described as:

"This data set includes descriptions of hypothetical samples corresponding to 23 species of gilled mushrooms in the Agaricus and Lepiota Family (pp. 500-525). Each species is identified as definitely edible, definitely poisonous, or of unknown edibility and not recommended. This latter class was combined with the poisonous one."



Names Dataset

Data Understanding

This Dataset contains 8124 completely nominal instances, 4208 (51.8%) classified as edible and the rest 3916 (48.2%) as poisonous. The rest 22 predictors refer to the characteristics of the mushroom, such as cap, bruises, odor, gill, stalk, veil, ring, spor, population and habitat.



Data Preparation

Almost all attributes are biased to the right, so data must be normalized and therefore converted to numerical in order to achieve a higher performant level. For this, "Nominal to Numerical" Rapidminer operator is used, followed by a "Normalize" one. Between them, "Principal Component Analysis" should be integrated to reduce the number of the resulting attributes (116). But prior all this, "Set Role" operator must be included to specify the label and missing values must be handled by an operator called "Replace Missing Values".

Modeling and Evaluation

Different models are builded to be able to choose the best one, based on the posterior evaluation. k-NN (k=5), Naive Bayes, SVM, Linear Regression and Decision Tree were used. Confusion Matrices are available in algorithm name links. The most performant algorithm appears to be k-NN (k=5).
